# 🖋️ HFR - Hierarchical Font Recognition (Skills & Architecture)

Этот документ описывает архитектуру, технические возможности и этапы развития системы поиска кириллических шрифтов по латинским аналогам.

## 🏗️ Архитектура системы

### 1. Нейросетевое ядро (ML Core)
- **Модель**: `HFRNet v2.1` на базе **EfficientNet-B0**.
- **Предобучение**: Используются веса **ImageNet**, что позволяет модели эффективно распознавать геометрические формы букв (засечки, изгибы, наклон).
- **Latent Space**: Модель проецирует изображение буквы в 256-мерное векторное пространство (embedding).
- **Нормализация**: Применяется **L2-Normalization** на выходе модели, что гарантирует стабильность поиска по косинусному сходству.

### 2. Поисковой движок (Vector Search)
- **Библиотека**: **FAISS** (Facebook AI Similarity Search).
- **Индекс**: `FlatL2` (точное сопоставление).
- **База данных**: Индексировано **4880 шрифтов** с поддержкой кириллицы.
- **Производительность**: Поиск по всей базе на видеокарте **RTX 3090** занимает менее 10мс.

### 3. Обработка изображений (Inference Engine)
- **Сегментация**: Автоматическое выделение букв через `OpenCV` (FindContours).
- **Padding 30%**: Буквы не обрезаются "в край", сохраняя мелкие типографические элементы.
- **Letterbox Resize**: Сохранение **Aspect Ratio** (узкие буквы остаются узкими). Это ключевое улучшение для различения Condensed и Extended шрифтов.
- **Polarity Fix**: Система работает в режиме "Черный текст на Белом фоне", что соответствует реальным пользовательским скриншотам.

## 🛠️ История улучшений (Changelog)

### Phase 2.1: Accuracy Fix [DONE]
- Переход от случайных весов к предобученным (ImageNet).
- Исправление инверсии полярности (теперь черное на белом).
- Точность на тесте `Georgia Pro` выросла до **97.6%**.

### Phase 2.2: Aspect Ratio [DONE]
- Внедрен пропорциональный ресайз. Модель научилась понимать "ширину" шрифта.
- Прекращено искажение букв при вписывании в квадрат 64x64.

### Phase 4: UX Polish [DONE]
- **Smart Labels**: Скрытие подписей полей через CSS (opacity 0 -> 1 при наведении).
- **Pagination**: Кнопка "Показать еще 5" (мгновенный поиск по ТОП-50 из памяти).
- **Multi-Source**: Поддержка Загрузки, Веб-камеры и Буфера обмена (Clipboard).

### Phase 4.1: Professional Interface Overhaul [DONE]
- **Clean UI**: Убраны все отвлекающие элементы (эмодзи), метки перенесены в полноценные заголовки.
- **Split Layout**: Внедрен "Студийный макет". Левая часть (настройки и оригинал) зафиксирована на экране через `position: sticky`, тогда как лента результатов справа скроллится независимо.
- **Typography Controls**: В разделе сегментации добавлены ползунки `Расстояние между букв` (letter spacing) и `Расстояние между слов` (word spacing). Эти настройки динамически перерисовывают превью каждого найденного шрифта!
- **Download Capability**: Под каждым результатом добавлена кнопка "Скачать шрифт", ведущая напрямую на физический файл на диске D.

### Phase 4.2: Polarity Auto-Detect [DONE]
- Внедрено авто-определение цвета фона (по медианной яркости). Если загружен скриншот с белым текстом на темном фоне, система инвертирует его "на лету" перед поиском контуров. Теперь сервис всеяден.

## 💡 Возможности и Ограничения
- **Возможности**: Подбор похожих по стилю (серифы к серифам, курсив к курсиву), поиск аналогов по весу (Bold/Light) и ширине.
- **Ограничения**: Требуется темный текст на светлом фоне для корректной сегментации (`cv2.threshold_otsu`).
